# MVP_010_FEATURES (Lite Only)

This document defines the **fixed feature set for the Lite plan MVP**.

The Lite MVP is designed as an **entry and activation experience**, not a daily or habit-forming product.

---

## Included Features (Lite MVP – Fixed)

### 1. Virtual Self Registration
- Users can create their virtual self.
- This is the entry point of all experiences.
- The goal is not perfect realism, but:
  - “This feels like me”
  - “I recognize myself in the screen”

---

### 2. Body Shape & Ratio Auto-Adjustment
- The system automatically adjusts body proportions and ratios.
- Users do **not** need perfect photos or scans.
- Based on existing body, ratio, and visual balance data.
- Purpose:
  - Reduce friction
  - Enable immediate immersion
  - Avoid setup fatigue

---

### 3. Unlimited Virtual Outfit Try-On
- Users can freely try unlimited outfits on their virtual self.
- This is the **core fun** of the Lite plan.
- No artificial restrictions on:
  - Categories
  - Styles
  - Number of tries

---

### 4. Real Brand / Real Item Only
- Only real, purchasable items are displayed.
- Each item includes:
  - Brand
  - Product name
  - Purchase link
- No fictional or placeholder items are allowed.
- Purpose:
  - Maintain trust
  - Ensure realism
  - Connect directly to real-world fashion

---

### 5. Outfit Tendency Indicator (No Blocking)

The system does **not** block or prohibit any outfit.

Instead, it displays **informational tendency labels** based on:
- Body shape
- Ratio
- Face balance (when available)
- Existing fashion theory and datasets

#### Tendency Labels (Fixed)

The system displays one of the following tendency labels:

- **Easy Fit**  
  “Likely to fit your body balance”

- **Neutral**  
  “Can work depending on styling choices”

- **High Difficulty**  
  “Technically possible, but requires balance adjustment”

These labels are **informational only**.  
Final decision always belongs to the user.

This follows professional stylist principles:
- No denial
- No forced judgment
- User autonomy is preserved

---

## Explicitly Excluded (Reserved for Standard / Pro)

The following features are **intentionally excluded** from Lite:

- Outfit history accumulation
- Daily or frequent recommendations
- Recommendation learning optimization
- Preference memory
- Event-specific styling (TPO)
- Weather or context logic
- Closet management
- Physical item delivery
- Habit formation mechanics
- Any Pro-only features

---

## Design Philosophy (Lite)

Lite is designed to deliver:

- Fun
- Self-recognition
- No failure feeling

But **not**:
- Daily dependence
- Full optimization
- Deep personalization

---

## Success Definition

If users say:

> “This is fun, feels like me, and I don’t fail —  
> but I want this every day.”

Then this MVP is considered successful.
